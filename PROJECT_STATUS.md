## 问卷调研关键结论（87份反馈，约一半为物理系学生）

### 现状与痛点
- 88%学生遇到不懂问题首选用AI（ChatGPT/Gemini等）
- 对现有AI最不满：瞎编不可信(58)、缺公式图表(37)、回答太泛不针对课程(27)、直接给答案没学会(21)、担心上传课件泄露(6)
- 最大学习困难：概念抽象(43)、理解了但不会做题(33)、公式混淆(33)、缺复习资料(31)、反馈滞后(29)

### 产品形态（强信号）
- **网页版是首选**（54票，远超手机端29/桌面端20/嵌入微信4）→ MVP必须是网页应用
- 设备习惯：电脑(64) > 平板(46) > 手机(26)

### 引导式学习（核心卖点已验证）
- 79/86认为引导式学习有帮助；1-10分平均意愿7.5分
- 最受欢迎方式（57票）："先帮我理清思路，等我说想看答案/尝试后再给答案"
- **重要：不能一刀切**——用户着急时（如考前/赶作业）需要能直接给答案，引导模式应可调节/场景感知，而非强制苏格拉底式
- 最需要场景：考前复习(64)、做作业遇难题(53)、预习(38)

### 附加功能期望（优先级供后续参考）
梳理知识体系(68) > 错题集/薄弱点分析(55) > 笔记+对话历史(54) > 匿名查看同学常见问题(26) > 社区讨论(14)

### 商业模式
免费(53)+免费含轻量推广(53) 远超买断(19)/订阅(13) → MVP不做付费墙

## 技术路线（已确认）

| 模块 | 选择 | 备注 |
|---|---|---|
| 知识库内容管理 | **Obsidian（免费）+ GitHub同步** | 个人/非商业用途免费，无需Obsidian Sync付费服务 |
| 向量库 | Chroma（本地Python库，非独立服务） | 免费，MVP够用 |
| Embedding模型 | 开源多语言模型（如 bge-small-zh / multilingual-e5），本地运行 | 中英文课件都能处理，零成本 |
| 后端 | Python + FastAPI | 一开始就写规范API，前端无论如何升级都不浪费 |
| LLM | **DeepSeek API（deepseek-v4-pro）** | 1M输入(cache miss) $0.435 / 1M输出 $0.87；自带thinking mode（默认开）。单次问答(~2000输入+500输出,cache miss)约$0.0013，团队5人×20次/天×30天≈$3.9 |
| 前端 | **Next.js + FastAPI（前后端分离，一步到位）** | 避免后期从Streamlit/Chainlit重写；MVP阶段功能保持最简 |
| 部署 | 本地开发先跑通，后续Render免费层 | 先验证逻辑 |

公式渲染：KaTeX/MathJax。图表识别等高级功能放到三阶段。

## MVP范围

**做（最小循环）**：
1. 选1-2门资料最全课程（量子力学/经典力学/电磁学票数最高）作为试点
2. 团队整理课程资料为Markdown
3. 脚本：切块 → embedding → 存入Chroma
4. 后端API：问题 → 检索 → 拼接prompt（含场景感知的引导逻辑）→ DeepSeek生成 → 返回回答+引用来源
5. 前端：聊天界面，支持公式渲染

**暂不做**：用户系统、错题集、社区、移动端、付费功能（留给三/四阶段）

## 未来想法（三阶段候选，非MVP）

- **多模型切换（V4-Pro + R1）**：根据问卷"引导需场景感知"的发现，设想在"烧脑解题"场景切换到推理能力更强的模型（如DeepSeek-R1）。
  - 当前评估：V4-Pro自带thinking mode，可能已能覆盖该需求，**应先用V4-Pro（含thinking mode）测试，验证是否够用**，再决定是否值得引入多模型路由的复杂度（场景识别、prompt适配、成本监控）
  - R1价格暂未核实，待需要时再查

### 已知问题 / 待改进
- ⚠️ **检索质量**：当前用 Chroma 默认向量模型（all-MiniLM-L6-v2，英文为主），对中文物理内容检索精度差（测试中"无限深势阱"问题没能召回对应段落）。**建议升级为多语言向量模型**：用 `fastembed`（基于 onnxruntime，无需 torch，轻量、可放 E 盘）加载 `bge-small-zh` 或 `multilingual-e5`。这是 MVP 跑通后的首要质量改进。
